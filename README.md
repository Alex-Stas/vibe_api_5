# Travel Wallet Bot

Telegram-бот на Python для учёта трат в поездках с конвертацией валют через `api.exchangerate.host`.

Бот позволяет:
- создавать несколько путешествий (кошельков) на пользователя;
- хранить баланс в валюте поездки и показывать эквивалент в домашней валюте;
- учитывать расходы из обычных сообщений с числом;
- хранить историю расходов в SQLite;
- менять курс вручную и сразу пересчитывать домашний эквивалент остатка.

## Технологии

- Python 3.10+
- `pytelegrambotapi`
- `requests`
- `python-dotenv`
- `pycountry` + `Babel` (ISO-справочник стран/валют)
- SQLite (локальная БД `travel_wallet.db`)

## Структура проекта

- `bot.py` — основной модуль Telegram-бота, бизнес-логика, работа с БД, меню, сценарии.
- `currency_api.py` — модуль доступа к API `https://api.exchangerate.host/convert`.
- `requirements.txt` — зависимости Python.
- `.env` — конфигурация (ключ API и токен бота).
- `travel_wallet.db` — локальная база данных (создаётся/используется автоматически).

## Конфигурация

Создайте файл `.env` в корне проекта:

```env
API_KEY=your_exchangerate_host_access_key
BOT_TOKEN=your_telegram_bot_token
```

## Установка

1. Создайте и активируйте виртуальное окружение:

```bash
python -m venv venv
```

Windows (PowerShell):

```powershell
.\venv\Scripts\Activate.ps1
```

2. Установите зависимости:

```bash
pip install -r requirements.txt
```

## Запуск

```bash
python bot.py
```

После запуска бот начинает polling Telegram API и готов к работе.

## Как работает бот

### Создание путешествия

1. Пользователь выбирает `Создать новое путешествие` (или `/newtrip`).
2. Вводит страну отправления и назначения.
3. Бот определяет валюты стран через ISO-справочник.
4. Получает текущий курс через `api.exchangerate.host/convert` и предлагает подтвердить.
5. Если курс не подходит, пользователь вводит свой курс вручную.
6. Пользователь вводит стартовую сумму в домашней валюте.
7. Бот конвертирует стартовую сумму через API и сохраняет начальный баланс.

### Учёт расходов

- Любое сообщение с числом считается расходом в валюте активного путешествия.
- Бот показывает пересчёт в домашнюю валюту и просит подтверждение (`✅ Да / ❌ Нет`).
- После подтверждения просит краткое описание (до 20 символов), сохраняет расход и уменьшает баланс.

### Баланс и курс

- Баланс показывается в двух валютах.
- Домашний эквивалент рассчитывается по текущему курсу.
- При изменении курса (`Изменить курс` / `/setrate`) домашний эквивалент остатка пересчитывается.

## Команды и меню

Inline-меню:
- `Создать новое путешествие`
- `Мои путешествия`
- `Баланс`
- `История расходов`
- `Изменить курс`

Slash-команды (альтернативно):
- `/start`
- `/newtrip`
- `/switch`
- `/balance`
- `/history`
- `/setrate`

## База данных

Используется SQLite с таблицами:
- `users` — пользователи Telegram;
- `trips` — путешествия/кошельки и их текущие настройки;
- `expenses` — история расходов по путешествиям.

## Описание модулей и функций

### `currency_api.py`

- `get_currency_rate(default, currencies)`  
  Возвращает набор курсов, последовательно вызывая конвертацию `1 <default>` в каждую валюту из списка.
- `convert_currency(amount, from_currency, to_currency)`  
  Выполняет запрос к endpoint `https://api.exchangerate.host/convert` с `access_key`, `from`, `to`, `amount`.

### `bot.py`

#### Работа с БД и пользователями
- `get_conn()` — создаёт SQLite-соединение.
- `init_db()` — создаёт таблицы `users`, `trips`, `expenses`.
- `ensure_user(user_id)` — регистрирует пользователя при первом обращении.
- `get_active_trip(user_id)` — получает активное путешествие пользователя.
- `set_active_trip(user_id, trip_id)` — переключает активное путешествие.
- `create_trip(...)` — создаёт новое путешествие и делает его активным.

#### Страны, валюты, расчёты
- `normalize_country(country)` — нормализация ввода страны.
- `build_country_index()` — строит индекс стран (ISO + локализованные названия + алиасы).
- `country_to_currency(country)` — определяет валюту страны по ISO-территории.
- `parse_amount(text)` — безопасный парсинг чисел из сообщений.
- `format_amount(amount)` — форматирование сумм для сообщений.
- `home_equivalent_by_rate(destination_balance, rate)` — пересчёт остатка в домашнюю валюту.
- `convert_with_api(amount, from_currency, to_currency)` — обёртка над API-конвертацией с обработкой ошибок.

#### UI и разделы бота
- `main_menu_markup()` — inline-меню.
- `persistent_menu_markup()` — постоянные reply-кнопки.
- `send_text(...)` — единая отправка сообщений с клавиатурой.
- `show_main_menu(chat_id, user_id)` — стартовый экран.
- `start_new_trip(chat_id, user_id)` — запуск сценария нового путешествия.
- `show_trips(chat_id, user_id)` — список путешествий и переключение.
- `show_balance(chat_id, user_id)` — текущий баланс в двух валютах.
- `show_history(chat_id, user_id)` — история последних расходов.
- `ask_set_rate(chat_id, user_id)` — ввод нового ручного курса.

#### Обработчики Telegram
- `cmd_start`, `cmd_newtrip`, `cmd_switch`, `cmd_balance`, `cmd_history`, `cmd_setrate` — slash-команды.
- `callback_router(call)` — обработка inline-кнопок.
- `text_router(message)` — основной обработчик текстов/состояний и учёта расходов.

## Возможные проблемы

- `API_KEY` или `BOT_TOKEN` не заданы в `.env` — бот завершится с ошибкой конфигурации.
- Нераспознанная страна — бот попросит ввести более стандартное название.
- Ошибка API/сети — бот покажет дружелюбное сообщение и предложит повторить позже.
